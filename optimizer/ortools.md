# OR-Tools 求解器：官方示例 vs. 实验室自动化场景对照指南

本指南将 OR-Tools 的核心求解器、数学优化模型，与两类场景进行 1 对 1 对照：

1. **官方示例 / 生活化示例**：优先采用 OR-Tools 官网文档中的经典示例，便于和官方教程、代码直接对应。
2. **合成生物自动化 / Biofoundry 场景**：将同一类优化问题映射到实验室自动化中的高通量调度、路径规划、板位布局、样本分配等问题。

## 二、求解器场景深度对照解析

---

### 1. CP-SAT Solver：复杂约束与调度优化

`CP-SAT` 适合处理大量整数变量、布尔变量、逻辑约束、时间约束和资源互斥约束。  
它尤其适合解决“多个任务、多个资源、复杂先后顺序、不能冲突”的问题。

---

#### 🏭 官方示例 / 生活化场景：车间作业调度 Job Shop Scheduling

OR-Tools 官网 CP-SAT 的经典示例之一是 **Job Shop Scheduling**。

##### 通俗解释

一家小型维修车间有多台机器，例如：

- 钻孔机
- 切割机
- 打磨机

现在有多个订单，每个订单都需要经过若干道工序，并且这些工序必须按固定顺序完成。

例如：

- 订单 A：先切割，再打磨
- 订单 B：先钻孔，再切割，再打磨
- 订单 C：先打磨，再钻孔

每台机器同一时间只能处理一个工序，目标是安排所有订单的加工顺序，使总完成时间最短。

##### 关键约束

- **工序先后顺序**：同一个订单的前一道工序完成后，下一道工序才能开始。
- **机器排他性**：同一台机器同一时间只能服务一个任务。
- **任务持续时间**：每道工序都有固定加工时长。
- **优化目标**：最小化所有订单的最终完工时间，也就是 makespan。

##### OR-Tools 实现方式

常用建模方式：

- 用 `model.NewIntervalVar()` 表示每道工序占用某台机器的一段时间。
- 用 `model.AddNoOverlap()` 保证同一台机器上的工序时间段不能重叠。
- 用 `model.Add(start_next >= end_prev)` 表示工序之间的先后关系。
- 用 `model.Minimize(makespan)` 最小化总完工时间。

---

#### 🔬 实验室自动化场景：高通量实验流程调度 HTS Workflow Scheduling

##### 对应关系

| 官方 Job Shop 示例 | 实验室自动化场景 |
| :--- | :--- |
| 订单 Job | 一个样本、一块微孔板、一个实验批次 |
| 工序 Task | 加样、混匀、孵育、离心、PCR、读板 |
| 机器 Machine | 移液工作站、PCR 仪、离心机、读板机、机械臂 |
| 工序时长 Duration | 每个实验步骤所需时间 |
| 工序先后关系 | 实验 SOP 中的流程顺序 |
| 机器不能同时处理多个任务 | 同一设备同一时间不能被多个板占用 |
| 最小化 makespan | 缩短整批实验完成时间，提高设备利用率 |

##### Biofoundry 应用示例

在一个高通量筛选流程中，每块 96 孔板需要依次经过：

1. 自动移液加样
2. 孵育 30 分钟
3. 加检测试剂
4. 避光反应 10 分钟
5. 读板机读取荧光信号

如果同时有几十块板进入系统，就会出现复杂的设备冲突和时间窗约束。

##### OR-Tools 实现方式

- 使用 `NewIntervalVar()` 定义每块板在每台设备上的占用时间。
- 使用 `AddNoOverlap()` 避免同一设备被重复占用。
- 使用先后约束表达 SOP 流程顺序。
- 如果某些步骤必须在特定时间窗内完成，例如“加药后 180 ± 10 秒读数”，可加入时间窗约束。
- 目标可以是：
  - 最小化总完成时间；
  - 最小化设备空闲时间；
  - 最大化单位时间处理板数；
  - 避免关键样本超时。

### 2. GLOP Linear Solver：连续变量线性规划

`GLOP` 适合求解所有变量都是连续实数、约束和目标函数都是线性的优化问题。

它常用于：

- 成本最小化；
- 配方优化；
- 营养配比；
- 资源分配；
- 线性生产计划。

---

#### 🥗 官方示例 / 生活化场景：Stigler Diet 饮食成本最小化

OR-Tools 线性规划中的经典官方示例是 **Stigler Diet Problem**。

##### 通俗解释

你希望用最少的钱搭配一份饮食方案，同时满足人体每天所需的基本营养。

可选食物包括：

- 牛奶
- 面包
- 鸡蛋
- 土豆
- 蔬菜
- 肉类

每种食物都有：

- 单价；
- 蛋白质含量；
- 热量；
- 钙含量；
- 维生素含量；
- 铁含量等。

目标是：  
在满足营养最低需求的前提下，让总花费最低。

##### 关键约束

- **连续变量**：每种食物购买多少，可以是连续数值。
- **营养下限**：蛋白质、热量、钙、维生素等必须达到最低标准。
- **成本目标**：最小化总成本。
- **线性关系**：总营养 = 各食物数量 × 单位营养含量的线性加和。

##### OR-Tools 实现方式

- 用 `solver.NumVar()` 定义每种食物的购买量。
- 用 `solver.Add()` 添加营养约束。
- 用 `solver.Minimize()` 最小化总价格。
- 用 `solver.Solve()` 求解最优饮食组合。

---

#### 🔬 实验室自动化场景：高通量培养基配方优化 Media Formulation Optimization

##### 对应关系

| Stigler Diet 示例 | 实验室培养基优化 |
| :--- | :--- |
| 食物 | 培养基组分，例如葡萄糖、氮源、盐、缓冲液、诱导剂 |
| 食物用量 | 每种试剂加入体积或浓度 |
| 营养需求 | 细胞生长所需碳源、氮源、微量元素、渗透压、pH 范围 |
| 总成本 | 培养基配制成本 |
| 最小成本饮食 | 最低成本且满足生长需求的培养基配方 |

##### Biofoundry 应用示例

在自动化培养基设计中，系统需要为多种菌株或细胞系自动生成培养基配方。

每种组分都可能影响：

- 生长速率；
- 产物表达量；
- 代谢负担；
- 渗透压；
- pH 缓冲能力；
- 实验成本。

##### OR-Tools 实现方式

- 用 `NumVar(0, upper_bound)` 定义每种培养基组分的体积或浓度。
- 用线性约束表达：
  - 总体积限制；
  - 碳源最低浓度；
  - 氮源最低浓度；
  - 盐浓度范围；
  - 成本预算上限。
- 用 `Minimize()` 最小化培养基成本，或用 `Maximize()` 最大化理论产量评分。

### 3. MIP Solver：混合整数线性规划

`MIP` 适合处理既包含离散选择，又包含线性数量关系的问题。

典型变量包括：

- `BoolVar`：是否选择、是否使用、是否分配；
- `IntVar`：选择多少个、使用第几个批次；
- `NumVar`：连续用量、成本、体积、时间等。

---

#### 📦 官方示例 / 生活化场景：Bin Packing 装箱问题

OR-Tools 官网 MIP 的经典示例之一是 **Bin Packing Problem**。

##### 通俗解释

你要搬家，有一堆物品需要装进箱子。每个箱子有最大承重，物品重量不同。

目标是：  
在不超过每个箱子承重的前提下，使用尽可能少的箱子装完所有物品。

##### 关键约束

- **0-1 变量**：物品 i 是否放入箱子 j。
- **容量约束**：每个箱子的总重量不能超过上限。
- **使用箱子变量**：箱子 j 是否被使用。
- **优化目标**：最小化使用的箱子数量。

##### OR-Tools 实现方式

常见变量：

- `x[i, j] = 1` 表示物品 i 放入箱子 j；
- `y[j] = 1` 表示箱子 j 被使用。

常见约束：

- 每个物品必须且只能放入一个箱子；
- 每个箱子的总重量不能超过容量；
- 如果物品放入箱子 j，则箱子 j 必须被标记为使用。

目标：

- `Minimize(sum(y[j]))`

---

#### 🔬 实验室自动化场景：实验样本批次与微孔板装载优化

##### 对应关系

| Bin Packing 示例 | 实验室自动化场景 |
| :--- | :--- |
| 待装箱物品 | 待检测样本、待筛选化合物、待执行实验任务 |
| 箱子 | 96 孔板、384 孔板、PCR 板、测序批次、仪器运行批次 |
| 箱子容量 | 孔位数量、反应体积、仪器通量、批次最大样本数 |
| 物品重量 | 样本占用孔数、所需反应体积、处理时间、成本 |
| 最少箱子 | 最少板数、最少批次数、最低运行成本 |

##### Biofoundry 应用示例

实验室每天接收大量样本，但每台仪器或每块板的容量有限。例如：

- 一块 96 孔板最多可放 96 个反应；
- 边缘孔可能要保留给水或 buffer；
- 阳性、阴性、空白对照必须占用固定孔位；
- 某些样本不能放在同一板上；
- 高优先级样本需要尽早进入批次。

##### OR-Tools 实现方式

- 用 `BoolVar` 表示样本是否被分配到某块板或某个批次。
- 用容量约束限制每块板的样本数量。
- 用逻辑约束处理不能同板、必须同板、必须包含对照等规则。
- 目标可以是：
  - 最小化使用板数；
  - 最小化批次数；
  - 最大化单板利用率；
  - 最小化高优先级样本等待时间。

##### 扩展：微孔板布局优化

如果进一步考虑具体孔位，例如边缘效应、重复组分散、对照孔固定位置等，则可以将变量扩展为：

- `x[sample, row, col] = 1` 表示样本是否放在指定孔位。

这类问题仍然可以用 MIP 或 CP-SAT 建模。

### 4. Routing Model：路径与访问顺序优化

`Routing Model` 是 OR-Tools 中专门用于路径规划的问题求解框架，适合：

- TSP：旅行商问题；
- VRP：车辆路径问题；
- CVRP：带容量车辆路径问题；
- VRPTW：带时间窗车辆路径问题；
- Pickup and Delivery：取送货问题。

---

#### 🚚 官方示例 / 生活化场景：Vehicle Routing Problem 配送车辆路径规划

OR-Tools Routing 官方文档中的核心示例是 **Vehicle Routing Problem, VRP**。

##### 通俗解释

一个配送中心有多辆货车，需要把包裹送到多个客户地址。

目标是：

- 每个客户都要被访问一次；
- 每辆车从配送中心出发；
- 可以回到配送中心；
- 总行驶距离或总行驶时间尽可能短。

如果加入容量限制，则变成 CVRP；  
如果加入客户收货时间窗口，则变成 VRPTW。

##### 关键约束

- **访问约束**：每个客户必须被访问一次。
- **路径连续性**：车辆路线必须前后连贯。
- **车辆数量**：可用车辆数量有限。
- **距离成本**：任意两个地点之间有距离或时间成本。
- **容量约束**：车辆不能超载。
- **时间窗约束**：客户必须在规定时间段内被服务。

##### OR-Tools 实现方式

- 用 `RoutingIndexManager` 管理节点索引。
- 用 `RoutingModel` 建立路径模型。
- 注册距离回调函数作为 `ArcCostEvaluator`。
- 添加 `Dimension` 表示容量、时间或距离。
- 设置搜索策略和局部搜索元启发式算法。
- 求解最短或最低成本路径。

---

| 最短路线 | 最快完成移液、转运或上下料 |

##### Biofoundry 应用示例

在自动化平台中，路径优化可用于：

- 移液枪头在 96 / 384 孔板之间的移动顺序；
- 声波点样仪在目标孔位之间的访问路径；
- 机械臂在培养箱、离心机、读板机之间的板转运路径；
- AMR 在自动化实验室中的耗材、样本、板架配送路线。

##### OR-Tools 实现方式

- 将孔位、仪器、板架位置抽象为节点。
- 用几何距离、移动时间或碰撞风险作为边成本。
- 使用 `RoutingModel` 求解访问顺序。
- 若多个机械臂或多台 AMR 同时工作，可建模为多车辆 VRP。
- 若实验步骤有时效要求，可加入时间窗维度。

### 5. Network Flows：高速图匹配与网络流

OR-Tools 的图算法适合结构清晰、规模较大、需要极快响应的问题。

典型问题包括：

- 线性指派问题 Linear Sum Assignment；
- 最大流 Max Flow；
- 最小费用流 Min Cost Flow；
- 二分图匹配；
- 供需网络分配。

---

#### 👷 官方示例 / 生活化场景：Linear Sum Assignment 工人任务指派

OR-Tools Graph 官方示例中常见问题是 **Linear Sum Assignment**。

##### 通俗解释

有若干名工人和若干项任务。  
每名工人完成不同任务的成本不同。

例如：

| 工人 | 任务 A | 任务 B | 任务 C |
| :--- | :--- | :--- | :--- |
| 工人 1 | 90 | 76 | 75 |
| 工人 2 | 35 | 85 | 55 |
| 工人 3 | 125 | 95 | 90 |

目标是：

- 每个工人最多分配一个任务；
- 每个任务分配给一个工人；
- 总成本最低。

##### 关键约束

- **一对一匹配**：工人与任务一一配对。
- **边权重**：某工人执行某任务的成本。
- **优化目标**：最小化总匹配成本。

##### OR-Tools 实现方式

- 使用 `SimpleLinearSumAssignment()`。
- 将工人和任务看作二分图两侧节点。
- 将成本作为边权重。
- 调用图算法快速求解最小成本匹配。

---

#### 🔬 实验室自动化场景：样本到孔位的极速分配 Sample-to-Well Assignment

##### 对应关系

| Linear Sum Assignment 示例 | 实验室自动化场景 |
| :--- | :--- |
| 工人 | 待处理样本、化合物、菌株、克隆 |
| 任务 | 微孔板孔位、仪器通道、检测批次 |
| 工人执行任务的成本 | 样本放入某孔位的风险、距离、优先级惩罚、稳定性评分 |
| 一对一匹配 | 每个样本占一个孔位，每个孔位放一个样本 |
| 最小化总成本 | 获得整体最优排板或分流方案 |

##### Biofoundry 应用示例

在流水线系统中，扫码枪不断读入样本，系统需要在极短时间内决定样本放入哪个孔位。

成本可以由多种因素组成：

- 急诊样本应放在更稳定的内圈孔；
- 边缘孔蒸发风险更高，关键样本应避免；
- 同类样本应均匀分散；
- 高优先级样本应优先靠近后续处理路径；
- 某些孔位已被对照占用。

##### OR-Tools 实现方式

- 用 `SimpleLinearSumAssignment()` 建立样本与孔位的二分图。
- 为每个“样本-孔位”组合计算成本。
- 禁止不允许的组合，或赋予极大惩罚成本。
- 在毫秒级完成最小成本匹配。

##### 何时改用 Min Cost Flow？

如果不再是一对一匹配，而是存在：

- 一个批次接收多个样本；
- 一个样本需要多个重复孔；
- 不同孔区有容量限制；
- 多个上游来源和多个下游目标；

则更适合使用 `SimpleMinCostFlow()` 建模为最小费用流问题。

### 6. Knapsack Solver：有限容量下的价值最大化

背包问题适合解决：

> 在容量、预算、重量、体积等资源有限的情况下，选择哪些项目，使总价值最大。

---

#### 🎒 官方示例 / 生活化场景：Knapsack Problem 背包装物

OR-Tools Knapsack 官方文档中的经典示例就是 **0-1 Knapsack Problem**。

##### 通俗解释

你准备去徒步旅行，背包容量有限。  
候选物品包括：

- 帐篷
- 睡袋
- 水壶
- 食物
- 相机
- 急救包
- 充电宝

每件物品都有：

- 重量；
- 价值或重要性。

目标是：

- 不超过背包最大承重；
- 选择总价值最高的一组物品。

##### 关键约束

- **0-1 决策**：每件物品要么带，要么不带。
- **容量限制**：总重量不能超过背包容量。
- **价值最大化**：选中物品的总价值最大。
- **可扩展为多维背包**：同时考虑重量、体积、预算等多个容量限制。

##### OR-Tools 实现方式

- 输入每件物品的价值 `values`。
- 输入每件物品在各个维度上的消耗 `weights`。
- 输入容量上限 `capacities`。
- 调用 `KnapsackSolver` 求解最优选择。

#### 🔬 实验室自动化场景：高通量化合物筛选库子集选择 Compound Library Subset Selection

##### 对应关系

| Knapsack 示例 | 实验室自动化场景 |
| :--- | :--- |
| 候选行李 | 候选化合物、菌株、突变体、sgRNA、引物组合 |
| 物品重量 | 实验成本、孔位占用、试剂消耗、处理时间 |
| 背包容量 | 微孔板孔位数、项目预算、仪器运行时间、试剂库存 |
| 物品价值 | 靶点覆盖度、结构多样性、预测活性、科学价值 |
| 选择最有价值物品 | 选择最值得上板实验的一批候选对象 |

##### Biofoundry 应用示例

在一次药物筛选中，候选化合物库可能有 5000 个分子，但本轮实验只能筛选 96 个或 384 个。

每个化合物都有：

- 采购成本；
- 溶解性风险；
- 结构新颖性；
- 靶点覆盖度；
- AI 模型预测活性；
- 是否已有类似化合物。

目标是：  
在预算和孔位有限的情况下，选择一组总体科学价值最高的化合物进入实验。（备注：如果单个化合物价值未知且也需优化，则选用贝叶斯优化）

##### OR-Tools 实现方式

- 用价值表示化合物的重要性或预测收益。
- 用重量表示成本、孔位占用、试剂消耗等。
- 设置容量为预算、孔位数量或实验时间。
- 使用 `KnapsackSolver` 选择最优子集。
- 若同时考虑预算、孔位、实验时长，可建模为多维背包问题。

## 三、求解器选择速查表

| 求解器 | 数学本质 | 官方示例 / 生活化场景 | 实验室自动化场景 |
| :--- | :--- | :--- | :--- |
| **CP-SAT** | 约束满足 + 整数优化 + 调度 | Job Shop Scheduling：车间工序与机器排程 | 高通量实验流程调度、设备占用、孵育时间窗、机械臂排班 |
| **GLOP** | 连续线性规划 LP | Stigler Diet：最低成本饮食配方 | 培养基配方优化、缓冲液配比、培养条件线性优化 |
| **SCIP / CBC / HiGHS** | 混合整数线性规划 MILP | Bin Packing：最少箱子装完所有物品 | 样本批次规划、板数最小化、仪器运行批次优化 |
| **Routing Model** | 路径优化 TSP / VRP | Vehicle Routing：配送车辆访问客户 | 移液路径、机械臂转运路径、AMR 实验室物流路线 |
| **Graph Algorithms** | 图论匹配 / 网络流 | Linear Sum Assignment：工人与任务一对一匹配 | 样本到孔位极速匹配、样本分流、批次容量分配 |
| **Knapsack Solver** | 容量约束下价值最大化 | Knapsack：背包容量有限时选择最有价值物品 | 化合物库子集选择、sgRNA 筛选、实验候选组合选择 |

## 四、建模选择建议

### 1. 如果问题主要是“时间怎么排”，优先考虑 CP-SAT

适合：

- 设备不能冲突；
- 工序有先后顺序；
- 任务有开始时间、结束时间、持续时间；
- 有严格时间窗；
- 需要处理复杂逻辑约束。

典型 Biofoundry 问题：

- 微孔板流水线调度；
- PCR / 读板机 / 离心机共享排程；
- 多机械臂协作；
- 孵育时间精确控制。

---

### 2. 如果问题是“连续比例怎么配”，优先考虑 GLOP

适合：

- 变量是连续值；
- 约束是线性的；
- 目标是成本最小或收益最大；
- 不涉及是否选择这类 0-1 决策。

典型 Biofoundry 问题：

- 培养基配方；
- 试剂浓度优化；
- 缓冲液配比；
- 理论代谢通量的线性近似。

---

### 3. 如果问题是“选不选 + 分多少”，考虑 MIP

适合：

- 有 0-1 选择；
- 有容量限制；
- 有批次、板、设备、人员等离散资源；
- 目标和约束可以写成线性形式。

典型 Biofoundry 问题：

- 样本分批；
- 微孔板使用数量最小化；
- 板位分配；
- 对照孔布局；
- 多实验任务合并运行。

---

### 4. 如果问题是“访问顺序怎么走”，使用 Routing Model

适合：

- 节点之间有距离或时间；
- 要规划访问顺序；
- 有一台或多台移动实体；
- 可能有容量和时间窗。

典型 Biofoundry 问题：

- 移液枪头移动顺序；
- 机械臂取放板路线；
- AMR 运输路径；
- 多仪器站点访问顺序。

---

### 5. 如果问题是“高速一对一匹配”，使用 Graph Assignment

适合：

- 一组对象匹配到另一组对象；
- 每个匹配有成本；
- 需要极快响应；
- 约束结构相对简单。

典型 Biofoundry 问题：

- 样本到孔位；
- 任务到设备；
- 板到读板机；
- 条码样本到检测通道。

---

### 6. 如果问题是“容量有限，挑哪些最值”，使用 Knapsack

适合：

- 候选对象很多；
- 每个对象有价值；
- 每个对象消耗资源；
- 总资源有限；
- 目标是价值最大化。

典型 Biofoundry 问题：

- 化合物子集选择；
- 突变体筛选；
- sgRNA 候选集合选择；
- 高价值样本优先上板。

---

## 五、一句话总结

| 问题特征 | 推荐求解器 |
| :--- | :--- |
| 复杂逻辑、先后顺序、时间窗、资源互斥 | **CP-SAT** |
| 连续变量、线性配方、成本最小化 | **GLOP** |
| 0-1 选择、批次规划、容量约束、线性目标 | **MIP / SCIP / CBC / HiGHS** |
| 多节点访问顺序、车辆路径、机械移动路径 | **Routing Model** |
| 一对一匹配、网络流、毫秒级分配 | **Graph Algorithms** |
| 容量有限下选择价值最高的一组对象 | **Knapsack Solver** |
